## Project Overview

This project contains five shell scripts that demonstrate practical Linux skills and the philosophy of open source software. Each script is fully commented and includes proper error handling.

- **Script 1** - Displays system information and identity
- **Script 2** - Checks if your chosen software is installed
- **Script 3** - Audits disk usage and file permissions
- **Script 4** - Analyzes log files for specific keywords
- **Script 5** - Generates a personalized open source manifesto

### Script 3: Disk and Permission Auditor

**Purpose:** Check disk usage and permissions of system directories

**Usage:**
```bash

```

**What it audits:**
- Critical directories: `/etc`, `/var/log`, `/home`, `/usr/bin`, `/tmp`, `/var`, `/boot`

### Script 4: Log File Analyzer

**Purpose:** Analyze log files by counting lines with specific keywords

**Usage:**
```bash
# Search for "error" (default keyword)
bash script4.sh /var/log/syslog

# Search for a specific keyword
bash script4.sh /var/log/syslog WARNING
bash script4.sh /var/log/auth.log FAILED
bash script4.sh /var/log/messages ERROR
```

**Common log files:**
- `/var/log/syslog` - System messages
- `/var/log/auth.log` - Authentication events
- `/var/log/messages` - General system messages
- `/var/log/daemon.log` - Daemon service logs
- `/var/log/kernel.log` - Kernel messages

**What it does:**
1. Opens the log file
2. Reads it line by line
3. Counts lines containing your keyword (case-insensitive)
4. Shows the last 5 matching lines for context

**Example:**
```bash
bash script4.sh /var/log/syslog error
```

**Example Output:**
```
Log File       : /var/log/syslog
Search Keyword : error (case-insensitive)

Total lines containing 'error': 42

Last 5 matching lines:
Mar 25 10:34:12 shubham kernel: [1234.567890] Error in module
Mar 25 10:45:23 shubham NetworkManager[1234]: <error> Failed to...
```

## Troubleshooting

### Problem: "Permission denied"
**Solution:**
```bash
chmod +x script1.sh script2.sh script3.sh script4.sh script5.sh
```

### Problem: Script not found
**Solution:** Make sure you're in the correct directory
```bash
cd ~/oss-audit
ls -la  # Should show your scripts
bash script1.sh
```

### Problem: Log file not found (Script 4)
**Solution:** Check available log files
```bash
ls -la /var/log/syslog /var/log/auth.log /var/log/messages
bash script4.sh /var/log/auth.log  # Use an available file
```

### Problem: Scripts won't execute
**Solution:** Verify they are valid shell scripts
```bash
file script1.sh  # Should say "Bourne-Again shell script"
bash script1.sh  # Run with bash explicitly
```
